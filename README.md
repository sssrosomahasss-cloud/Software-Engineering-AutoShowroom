# Технічне завдання на розробку програмного продукту «AutoDrive Digital»

## 1. Найменування та область застосування
**Найменування:** Програмний продукт «AutoDrive Digital» — вебсистема для автоматизації роботи автосалону.
**Область застосування:** Використовується автодилерами для представлення автопарку та управління запитами клієнтів.

## 2. Підстава для розробки
Розробка проводиться в межах ЛР №10-11 з дисципліни «Основи програмної інженерії».
**Стандарти:** ISO/IEC 25010, Agile Manifesto, Scrum Guide 2020.

## 3. Призначення розробки
* **Функціональне:** цифровізація каталогу авто, онлайн-запис на тест-драйв, порівняння характеристик.
* **Експлуатаційне:** доступ через браузери (Desktop/Mobile) 24/7.

## 4. Технічні вимоги

### 4.1. Склад продукту
* Модуль "Каталог автомобілів".
* Модуль "Конфігуратор" (вибір кольору/дисків).
* Модуль "Бронювання та Тест-драйв".
* Адміністративна панель для менеджера.

### 4.2. Функціональні характеристики (ЩО робить система)
1.  **Пошук:** фільтрація за маркою, ціною та типом палива.
2.  **Порівняння:** зіставлення параметрів двох моделей.
3.  **Конфігуратор:** інтерактивна зміна візуальних параметрів авто.
4.  **Форма зв'язку:** онлайн-запис на тест-драйв із валідацією телефону.
5.  **Адмінка:** додавання, видалення та редагування карток авто.

### 4.3. Нефункціональні вимоги (ЯК працює система)
1.  **Час відгуку:** завантаження сторінки не більше 2 секунд (Performance).
2.  **Адаптивність:** коректне відображення на смартфонах (Usability).
3.  **Безпека:** захист персональних даних через протокол HTTPS (Security).

### 4.4. Технічні засоби
* **Стек:** HTML5, CSS3, JavaScript (React), Node.js, PostgreSQL.
* **Браузери:** Chrome 110+, Safari 15+.

## 5. Стадії розробки (Agile Scrum)
* **Спринт 1:** Аналіз вимог та дизайн (2 тижні).
* **Спринт 2:** Розробка бази даних та каталогу (2 тижні).
* **Спринт 3:** Функціонал бронювання та конфігуратор (2 тижні).
* **Спринт 4:** Тестування та виправлення помилок (1 тиждень).

## 6. Порядок контролю та приймання
* Тестування: Unit-тести, UI-тести, перевірка швидкості завантаження.
* Критерій приймання: 100% виконання функцій та відсутність помилок у консолі.
